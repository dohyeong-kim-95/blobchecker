# Blobchecker 개발 저널

## 문제 정의

50×200 픽셀 격자에서 8개의 이진 블롭(blob) 레이어를 재구성한다.
매 이터레이션마다 좌표 (row, col) 하나를 쿼리하면 8개 레이어 전체에 대한 레이블(0 또는 1)이 동시에 반환된다.
예산: 총 1,500번 쿼리 (전체 10,000 픽셀의 15%).

**채점 기준 (두 조건 모두 충족해야 pass)**
- 각 레이어 픽셀 정확도 ≥ 98%
- 예측된 블롭의 높이·너비가 실제 블롭의 ±5% 이내

**주요 비대칭**: 오라클은 `truth_full_mask` (블롭 + 아웃라이어)를 반환하지만, 채점은 `truth_blob_mask` (블롭만)로 한다. 아웃라이어 픽셀을 블롭으로 잘못 예측하면 정확도가 떨어진다.

---

## 1단계: Pre-planned Greedy Coverage (베이스라인)

### 접근법
모든 1,500개 쿼리 좌표를 실행 전에 미리 결정한다.
탐욕 알고리즘으로 공간 커버리지를 최대화한다:
- 매 단계마다 아직 커버되지 않은 픽셀을 가장 많이 추가하는 좌표를 선택
- fftconvolve를 이용해 O(H·W·log(H·W)) 속도로 계산

**재구성**: 행 단위 스캔라인 채우기
- 각 행에서 가장 왼쪽~오른쪽 양수 관측값 사이를 1로 채움
- 내부의 관측된 0 주변을 지움 (구멍 억제)
- 관측값 없는 행은 가장 가까운 관측 행에서 복사

### 결과 (10시드 평균)
```
mean_min_accuracy: 0.9279  |  시간: 7.0s/시드  |  pass: 0/10
```

### 부족한 점
- 쿼리 간격이 행 방향 ~5픽셀, 열 방향 ~10픽셀 → 경계 불확실성 ±5-10픽셀
- 경계에 직접 쿼리가 없어 h_pred < h_truth 자주 발생
- 구멍(hole)은 쿼리가 우연히 내부에 들어갈 때만 감지됨
- 재구성에서 정확히 관측된 행 경계만 사용하므로 오차 ~7%

## 2단계: Geometry-First Adaptive (적응적 알고리즘)

### 핵심 아이디어
쿼리를 미리 결정하는 대신, 매 단계마다 "가장 불확실한" 픽셀을 선택한다.

**Phase 1 — 거친 탐색 (stride 5×10, ~200 쿼리)**
스트라이드 격자로 모든 8개 레이어의 블롭 위치를 대략적으로 파악한다.

**Phase 2 — 적응적 엔트로피 취득 (나머지 ~1,300 쿼리)**
믿음(belief) 텐서 기반으로 각 픽셀의 불확실성을 계산:
```
score(r,c) = Σ_k [1 - 4·(p_k(r,c) - 0.5)²]
```
가장 점수가 높은 (= 가장 불확실한) 미쿼리 픽셀을 선택한다.

**믿음 전파**: 쿼리 후 주변 (2R+1)×(2R+1) 이웃에 Chebyshev 거리 기반 감쇠로 업데이트:
```
p[:, r, c] ← p[:, r, c]·(1-w) + labels·w,   w = 0.65^dist
```

이 전파의 효과:
- 블롭 내부 → 믿음 → 1 → 엔트로피 → 0 → 더 이상 쿼리 안 함
- 블롭 외부 → 믿음 → 0 → 엔트로피 → 0 → 더 이상 쿼리 안 함
- **경계** → 믿음 ≈ 0.5 → 엔트로피 → 1 → 자연스럽게 경계에 집중

**예측**: 믿음 ≥ 0.5 임계값 + 최대 연결 컴포넌트(CC) 필터

### 결과 (10시드 평균)
```
mean_min_accuracy: 0.9648  |  시간: 1.4s/시드  |  pass: 0/10
```

**베이스라인 대비 개선**: +0.037 정확도, 5배 빠름

### 남은 문제
**h_pred > h_truth** (높이 과대추정)가 체계적으로 발생한다.

원인: 경계에서 distance=2 떨어진 외부 픽셀의 믿음이 단일 양수 관측으로 올라간다:
```
w = 0.65² = 0.4225
p_new = 0.4 × (1-0.4225) + 1 × 0.4225 = 0.655  >  0.5 threshold
```
즉, 블롭 경계 바깥 2픽셀까지 "블롭"으로 예측되어 높이가 2-6행 과대추정된다.

## 시도한 수정들 (실패한 실험들)

### 실험 1: Scanline 재구성으로 교체

**아이디어**: `predict()`에서 믿음 임계값 대신 관측값 기반 스캔라인을 사용하면 과대추정이 없어질 것.

**결과**: `mean_min_accuracy 0.8869` (훨씬 나빠짐!)

**원인**: 적응적 쿼리는 경계에 집중하기 때문에, 블롭 극단부(상단/하단) 행들이 쿼리되더라도 해당 행의 블롭 범위 전체를 커버하지 못한다. 스캔라인은 관측된 행에서 가장 왼쪽~오른쪽 양수 관측값만 사용하므로, 좁은 극단부 행은 심하게 과소추정된다.

**교훈**: 스캔라인 재구성은 **균일한 쿼리 분포**에만 적합하다. 적응적 알고리즘에는 믿음 전파 기반 예측이 필수다.

### 실험 2: 관측 제약 보정

**아이디어**: 어떤 행이 쿼리되었는데 해당 레이어에서 양수가 전혀 없다면, 그 행을 강제로 0으로 설정한다.

**결과**: `mean_min_accuracy 0.9536` (더 나빠짐)

**원인**: 블롭의 극단부 행(top/bottom)이 좁아서 거친 탐색 격자에서 양수를 받지 못한 경우, 이 행이 잘못 제거된다. h_pred가 h_truth보다 7행이나 적게 추정되는 경우 발생 (seed 2, layer 0).

**교훈**: 관측 없이 외부 행과 좁은 블롭 극단 행을 구분할 수 없다. 이 수정은 오히려 undershoot를 만든다.

### 실험 3: 비대칭 전파 (양수만 전파)

**아이디어**: 음수 관측(label=0)이 경계에서 블롭 내부로 전파되면 내부 믿음이 0.5 아래로 떨어진다. 양수만 전파하면 내부 믿음이 안정적으로 유지될 것.

**결과**: `mean_min_accuracy 0.7818` (대폭 악화!)

**원인**: 거친 탐색 격자의 열 간격이 10픽셀이지만 PROP_RADIUS=3이다. 쿼리된 열에서 3픽셀 이상 떨어진 내부 블롭 픽셀들은 양수 전파를 받지 못하고 사전 확률 0.4에 머문다. 이 픽셀들이 모두 "빈 공간"으로 예측된다.

더 나빠지는 이유: 음수 전파가 없으면 외부 픽셀들의 믿음이 줄지 않아 엔트로피가 높게 유지된다. 적응적 쿼리가 외부 픽셀을 계속 쿼리하며 예산을 낭비한다.

**교훈**: 대칭 전파가 필수다. 음수 전파는 (a) 외부 픽셀의 불확실성을 줄여 예산을 효율적으로 쓰게 하고, (b) 적응적 쿼리가 외부 대신 경계로 향하게 안내한다. 내부 믿음이 약간 떨어지더라도 전체적으로 훨씬 유익하다.

## 현재 상태 및 분석

### 최종 벤치마크 결과
```
알고리즘          pass    mean_min_acc  시간
preplanned        0/10    0.9279        7.0s
geometry_first    0/10    0.9648        1.4s
```

### 왜 98%에 도달 못하는가?

**블롭 경계 픽셀 수**: 평균 블롭 약 35×150 픽셀 → 경계 ~370 픽셀/레이어
**현재 오차**: 레이어당 ~352 픽셀 오류 (1500/0.9648× ... )
→ 오차의 대부분이 경계 픽셀에 집중

**예산 제약**: 1,500 쿼리 / 10,000 픽셀 = 15% 커버리지
8개 레이어 × 경계 ~370 픽셀 = 2,960 경계 픽셀
1,500 쿼리가 모든 경계를 커버하기엔 부족

**아웃라이어 영향**: 아웃라이어가 블롭 크기의 ~5%, 50% 확률로 존재
→ 예상 아웃라이어: ~130 픽셀/레이어
→ CC 필터가 독립 아웃라이어는 제거하지만, 경계 근처 아웃라이어는 메인 블롭과 합쳐져 남음
→ 이론적 정확도 상한 ~98-99%

**h_pred > h_truth 문제**: distance=2에서의 믿음 전파가 0.655 (임계값 0.5 초과)
→ 매 시드마다 높이 2-6행 과대추정
→ h_pass 거의 항상 실패

### 정확도 수렴 패턴

적응적 쿼리 후 초반 ~500 이터레이션에서 급격히 상승:
```
iter 100: ~0.50  (거친 탐색 중)
iter 300: ~0.78  (거친 탐색 완료)
iter 500: ~0.96  (적응 단계 효과 발현)
iter 1500: ~0.965 (점진적 개선 후 안정화)
```

500 이터레이션 이후 수렴 → 경계 픽셀 재구성의 근본적 한계 반영

## 다음 개선 방향

### 단기 (가능성 높음)

1. **탐색 격자 최적화**: COARSE_STEP_C를 10→8로 줄이면 3열 간격 이상에서 전파 도달 범위 안으로 들어온다 (PROP_RADIUS=3). 이로써 내부 블롭 픽셀의 초기 커버리지가 향상된다.

2. **전파 반경 증가**: PROP_RADIUS=3 → 4, PROP_DECAY 조정. 더 넓은 반경으로 거친 탐색 후 더 좋은 초기 믿음을 제공할 수 있다.

3. **높이 수렴 전략**: Phase 2에서 현재 예측된 블롭의 상단/하단 경계 행을 명시적으로 타겟팅하는 보조 쿼리 전략 추가.

### 중기 (탐색 필요)

4. **아웃라이어 감지**: 관측된 양수 중 인근에 다른 양수가 없는 고립된 점들을 식별하여 전파에서 제외한다. 블롭 크기 기반 믿음 우선순위가 도움이 될 수 있다.

5. **Level-set estimation**: 블롭 내부/외부 경계를 직접 추적하는 straddle 휴리스틱 기반 접근. 단, 8개 레이어 동시 처리에 적합하게 변형 필요.

6. **엔트로피 가중 재구성**: predict()에서 높은 믿음(>0.7)의 픽셀만 블롭으로 예측하여 경계 오버슈팅 감소. 단, 내부 unobserved 픽셀에 대한 믿음 강도 확인 필요.

### 변경 사항

**파라미터 조정:**
- `COARSE_STEP_C`: 10 → 8 (stride 8 + PROP_RADIUS 4 → 모든 내부 픽셀이 전파 반경 내)
- `PROP_RADIUS`: 3 → 4 (쿼리당 더 넓은 커버리지)

**Trim 재구성 추가:**
- predict()에서 CC 필터 후, 예측 블롭의 top/bottom 행이 confirmed-zero (label=0 관측만 있고 label=1 없음)면 제거
- h_pred > h_truth 오버슈팅을 직접 수정하는 핵심 메커니즘

### 실패한 실험 (Phase 2 방향 오류)

**실험 4: Phase 1.5 스캔 방향 오류 (BOUNDARY_BAND=2, ±2 around detected top)**
- 결과: 0.9642 (악화)
- 원인: ±2 스캔은 블롭 내부를 스캔 → label=1 전파가 역효과. detected top 위쪽 (외부) 행만 스캔해야 함.

**실험 5: 외부 스캔 방향은 맞지만 레이어 순차 처리 (BOUNDARY_SCAN_STEP=5, cap=200)**
- 결과: 0.9679 (개선)
- 문제: cap=200으로 레이어 0의 경계만 커버, 레이어 1-7은 스캔 미흡 → 일부 레이어 trim 작동 안함

### 남은 문제

**정확도 98% 미달**: 현재 ~96.9%, 목표 98%.

98% = 레이어당 ≤200개 오류 픽셀 필요. 현재 ~305개 오류. 줄여야 할 픽셀: ~105개.

예상 오류 원인:
1. 열 방향 경계 불확실성: ~2픽셀 × 블롭 둘레 ~520픽셀 = ~100픽셀 오류
2. 경계 근처 아웃라이어 merge: CC 필터로 제거 안 되는 연결된 아웃라이어 ~50픽셀
3. h/w 잔여 오차: ~30픽셀

**수렴 패턴 변화**: Phase 2 추가 이후 초반(iter 300-500)에서 정확도 더 빠르게 상승:
```
iter 100: ~0.55  (coarse 탐색 중)
iter 300: ~0.93  (Phase 2 완료)
iter 500: ~0.95  (entropy 단계 효과)
iter 1000: ~0.97
iter 1500: ~0.97 (수렴)
```

Phase 3 예산(~1050 쿼리)이 열 방향 경계 개선을 위해 모두 사용되지만 빡빡함.

## 핵심 교훈

1. **쿼리 전략과 재구성은 커플링되어 있다**: 스캔라인 재구성은 균일 커버리지가 전제. 적응적 쿼리에는 믿음 기반 예측이 필수.

2. **대칭 전파가 중요하다**: 음수 전파는 (a) 외부 영역 불확실성을 낮춰 예산 낭비를 막고, (b) 경계 방향으로 쿼리를 유도하는 핵심 역할을 한다. 비대칭 전파는 처참한 결과를 낳는다.

3. **15% 예산은 경계 재구성에 빡빡하다**: 8개 레이어 × ~370 경계 픽셀 = 2,960 경계 픽셀에 1,500 쿼리만 사용 가능. 내부나 외부 픽셀에 사용된 쿼리는 경계 정확도에 직접 기여하지 않는다.

4. **CC 필터의 효과**: 고립된 아웃라이어 픽셀을 제거하여 정확도를 크게 향상시킨다. 특히 pre-planned 베이스라인에서 h_pred > h_truth 문제를 해결하는 데 결정적이었다.

5. **적응적 알고리즘은 5배 빠르다**: fftconvolve 사전계획(7s/시드) vs 믿음 전파 적응(1.5s/시드). 계산 효율도 실질적 이점이다.

6. **Trim 재구성의 핵심**: 경계 행의 confirmed-zero 관측(label=0만 있고 label=1 없음)을 이용해 전파 오버슈팅으로 생긴 가짜 행을 직접 제거. h_pred > h_truth 문제를 근본적으로 해결.

7. **라운드로빈 경계 스캔의 필요성**: 순차 레이어 처리 + 작은 cap은 첫 레이어만 커버. 라운드로빈으로 8개 레이어 균등 커버해야 trim이 모든 레이어에서 작동함.

8. **Phase 2 스캔 방향**: detected top_r의 위쪽 (외부 영역)을 스캔해야 confirmed-zero를 얻을 수 있음. ±2 주변 스캔은 내부도 포함해 역효과.
